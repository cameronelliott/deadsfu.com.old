Docker#

Docker is a great way to use DeadSFU. If you are new to Docker, you might find it easier to learn DeadSFU with plain-old binaries. But, we offer both ways to use DeadSFU, so it’s up to you.

ðŸ’¡
There is nothing to install to use DeadSFU with Docker. But you must have a system with Docker already installed.
ðŸ’¡
Docker is tightly coupled with the Linux kernel. For performance reasons we recommend only using DeadSFU with Docker on Linux systems.

Compile from Source#

There are two repos you need to compile from source: deadsfu and deadsfu-binaries. deadsfu-binaries must be nested inside deadsfu when compiling.

You don’t need archane git tools like git lfs or git submodules.

You need a version of Go greater than 1.16, we recommend 1.16.5 or later.

Clone the main repo:

git clone https://github.com/x186k/deadsfu.git

Change dir:

cd deadsfu

Clone the binaries repo:

git clone https://github.com/x186k/deadsfu-binaries.git

Build with Go:

go build .
