Surprisingly Easy

We want SFU1 to be surprisingly easy to use. We go to great lengths to achieve this. Let us know how we are doing.

Massively Scalable

SFU1 has been designed from the ground-up to be multi-server scalable, and for implementation to also be as easy as possible.

Broadcasting Ready

Real-time audio, video broadcasting is a sweet-spot for SFU1. You can create multi-server broadcasting systems with a few Docker instances. Auto-scaling systems are possible too.

Cloud Native

We encourage using SFU1 with Docker, because itâ€™s a great way to build multi-server systems. Of course you can use binaries or build from source if you prefer.

No Coding Necessary

An HTTP server and a simple HTML/JS broadcast and watch application is built in. You can turn this off if you decide to use your own HTML/JS.

Simulcast and Multi-track Switching

SFU1 supports 10 video tracks and one audio track on publisher ingress. Subscribers can receive a single video track and ask the SFU to switch between sources. More than 10 tracks can be enabled using a command argument.
